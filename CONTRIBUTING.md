# Contributing to procx

Thank you for your interest in contributing to procx!

## Development Setup

1. Fork and clone the repository
2. Install dependencies: `npm install`
3. Build the project: `npm run build`
4. Start development: `npm run dev`

## Making Changes

1. Create a feature branch: `git checkout -b feature/your-feature-name`
2. Make your changes following the existing code style
3. Run linting: `npm run lint:fix`
4. Build and test: `npm run build`
5. Commit with a descriptive message
6. Push and create a Pull Request

## Code Standards

- Use TypeScript for all new code
- Follow existing patterns and naming conventions
- Run `npm run lint:fix` before committing
- Ensure cross-platform compatibility (macOS, Linux, Windows)

## Pull Request Checklist

- [ ] Code builds without errors
- [ ] Linting passes
- [ ] Documentation updated if needed
- [ ] Descriptive commit messages

## Bug Reports

Include:
- Operating system and Node.js version
- Complete error message and steps to reproduce
- Expected vs actual behavior

## Security

For security vulnerabilities, please use GitHub Security tab or contact maintainers directly instead of opening public issues.
